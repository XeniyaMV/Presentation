What?

They are patterns used to match character combinations in strings.

Why?

They provide a concise and flexible way to search, extract, and manipulate text in JavaScript.

Basic Syntax

Creating a regular expression

  • Using a regular expression literal, which consists of a pattern enclosed between slashes 
    									
										const re = /ab+c/;
  • Calling the constructor function of the RegExp object 
    									
										const re = new RegExp("ab+c");

How to use?

Regular expressions are used with the RegExp methods.

RegExp.prototype.test() is a method that checks for a match in a string. It returns true or false

							
								function isValidEmail(email) {
									const emailPattern = /@/;
									return emailPattern.test(email);
								}
								
								console.log(isValidEmail('123')); // false
								console.log(isValidEmail('abc@abc')); // true
								console.log(isValidEmail('@abc@')); // true

Special characters in regular expressions

  • x* - Matches the item "x" 0 or more times.

								function isValidEmail(email) {
									const emailPattern = /^[a-zA-Z0-9._%+-]+@/;
									return emailPattern.test(email);
								}
								
								console.log(isValidEmail('123')); // false
								console.log(isValidEmail('abc12@abc')); // true
								console.log(isValidEmail('@abc@')); // false

Integration with string methods

Regular expressions can be integrated with String methods: match(), matchAll(), replace(), replaceAll(), search(), and split().

Advanced searching with flags

  • i - With this flag the search is case-insensitive: no difference between A and a.
  • g - With this flag the search looks for all matches, without it - only the first match is returned.
  • s - Allows ' . ' to match newline characters.
  • u - treat a pattern as a sequence of Unicode code points.

Example

							
								const str = 'We will, we will rock you';
								console.log(str.match(/we/ig)); // [We, we]
